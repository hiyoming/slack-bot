# 🏥 병원 마케팅 관리용 Slack 에이전트 (1단계)

이 프로젝트는 70여 개 병원의 마케팅을 전담 관리하기 위해 모듈화된 아키텍처로 설계된 Slack 에이전트입니다.

## 📂 프로젝트 구조
- `index.js`: 모든 슬랙 이벤트를 수신하고 담당 모듈로 전달하는 라우터
- `channels/`: 각 채널별 전용 로직 폴더 (현재 `design.js` 1단계 구현)
- `data/`: 사내 인트라넷 연동 전 테스트를 위한 목업(JSON) 데이터 폴더
- `utils/`: DB 연동, 슬랙 통신 등 공용 유틸리티 폴더

---

## 🛠️ 1단계 테스트 준비하기

### 1. 슬랙 채널 ID 확인하기
봇이 `#디자인` 채널의 메시지만 처리하도록 하려면 해당 채널의 고유 ID를 알아야 합니다.
1. 슬랙 PC 버전에서 `#디자인` 채널에 들어갑니다.
2. 상단의 **채널 이름**을 클릭해서 정보 창을 엽니다.
3. 정보 창 맨 아래에 있는 **채널 ID (예: C01234ABCDE)** 를 복사합니다.

### 2. `.env` 파일 업데이트
기존에 있던 `.env` 파일을 메모장으로 열고, 방금 복사한 채널 ID를 추가합니다.
```env
SLACK_BOT_TOKEN=xoxb-기존토큰그대로
SLACK_SIGNING_SECRET=기존시크릿그대로
DESIGN_CHANNEL_ID=복사한_채널_ID
```

### 3. 슬랙 [Interactivity(버튼)] 기능 켜기
디자인 완료 시 발송되는 "승인/반려" 버튼을 작동시키려면 슬랙 앱 설정이 하나 더 필요합니다.
1. [Slack 앱 관리 페이지](https://api.slack.com/apps) 에서 내 앱을 클릭합니다.
2. 왼쪽 메뉴에서 **[Interactivity & Shortcuts]** 를 누릅니다.
3. 스위치를 **On** 으로 켭니다.
4. `Request URL` 칸에 기존 Event URL과 똑같은 주소에서 끝에만 바꿔서 넣습니다.
   - 기존: `https://.../slack/events`
   - **입력할 주소:** `https://.../slack/interactions`
5. 화면 아래의 **[Save Changes]** 를 누릅니다.

## 🚀 1단계 테스트 방법

### 서버 실행
터미널(cmd)을 열고 프로젝트 폴더에서 아래 명령어를 실행합니다.
```bash
node index.js
```

### 테스트 시나리오
1. **디자인 요청하기**
   - 슬랙 `#디자인` 채널에 들어가서 아래와 같이 입력합니다. (단어 띄어쓰기 주의)
   - `[강남OO의원] 인스타카드뉴스 - 여름 이벤트용 이미지 제작 부탁드립니다.`
2. **완료 처리하기**
   - 위에서 작성한 메시지에 **스레드(댓글)**를 답니다.
   - 봇을 멘션하며 완료를 알립니다. 예: `@봇이름 완료되었습니다!`
3. **결과 확인**
   - **(승인필요 항목 테스트):** `인스타카드뉴스`는 승인이 필요한 항목이므로, 담당자(현재는 `U01234567`로 목업되어 있으니 에러가 나거나 안 올 수 있습니다. `data/hospitals.json`을 열어서 `manager_slack_id`를 회원님 본인의 슬랙 ID로 수정하세요!)에게 DM으로 승인 버튼이 날아옵니다.
   - **(안내만 항목 테스트):** `서브페이지제작`으로 똑같이 테스트해보면 버튼 없이 완료 안내 DM만 날아옵니다.

> 💡 **본인 슬랙 ID 찾는 법:** 슬랙 왼쪽 아래 본인 프로필 사진 클릭 -> 프로필 보기 -> 점 3개 메뉴 클릭 -> '멤버 ID 복사' (U로 시작하는 영문/숫자 조합)
